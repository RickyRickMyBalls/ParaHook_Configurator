PARA-HOOK CONFIGURATOR — PROJECT CONTEXT
Project

ParaHook Configurator

A node-based CAD system with a deterministic graph architecture and 3D preview.

Inspired by:

Blender Geometry Nodes

Houdini

Grasshopper

But designed with deterministic compile + worker CAD runtime.

Core Architecture
Graph
  ↓
Resolver
  ↓
Compiler
  ↓
Worker CAD Runtime
  ↓
Viewer
Key rule
Evaluation / selector / render / compile
MUST NOT mutate graph topology.

Only:
graphCommands
normalizeGraphForStoreCommit

may mutate topology.

Major Systems
Spaghetti Editor

Node-based editor for graph creation.

Nodes contain:

Drivers
Inputs
Feature Stack
Outputs
Drivers

Drivers are param nodes pinned to a part node.

Ports:

in:drv:<paramId>
out:drv:<paramId>

Drivers can be wired to anything.

Numeric drivers support offset mode:

effectiveValue = drivenValue + offsetValue

Offset applies only to output/UI, not compute semantics.

OutputPreview

Special system node controlling preview rendering.

Pipeline:

PartNode.solid
      ↓
OutputPreview slot
      ↓
build()
      ↓
meshShape()
      ↓
Viewer

OutputPreview maintains invariant:

exactly one trailing empty slot
Important Architectural Rule
Graph topology never mutates during evaluation.

Only these mutate topology:

graphCommands
normalizeGraphForStoreCommit
